Mi ez a HTML a javascriptünkben?

A JSX az HTML beleépítve a JS-be:

if (score > 100) {
  return <b>You win!</b>
}
if (score > 100) {
  return <b>You win, { playerName }</b>
}

(A JS változót keresi aminek a neve playerName)

JSX használata

  • A JSX az nem valid JavaScript
    • Át kell lennie ''transpileolva'' JavaScriptté
  • A Babel erre jó

Transpielolás a böngészőben

  • Könnyű elkezdeni, semmit nem kell feltelepítenünk

  • Töltsük be az önálló Babel könyvtárat:

    <script
  src="https://unpkg.com/babel-standalone"></script>

  • Jelöljük meg a JSX-et type="text/jsx":

    <script src="index.js" type="text/jsx"></script>

  • Olvassuk el a dokumntációt, hogy hogyan használjuk a command line-ban

Babel használata command line-ban

Könnyű JSX-et JS-té transpileolni ilyen módon a böngészőben, azonban ez nem kifizetődő valódi applikációk keszítéséhez: Kell kb 1 másodperc a lefutási időhöz, tehát lassú lenne, és kényelmetlen a felhasználóknak.

Sokkal jobb ezt a command line-ban 1x megcsinálni és később felhasználni a már készen lévő JS fájlokat.

Ehhez a következőket kell tenni:

  1. npm telepítése

  2. Babel React-hoz való telepítésének parancsa

       $ npm install @babel/core @babel/cli @babel/preset-react

  3. Egy fájl átkonvertálása:

    $ node_modules/@babel/cli/bin/babel.js --presets @babel/react
     file.jsx > file.js

Több top-level komponens renderelése

A React 16-os verziója előtt nem volt lehetséges több komponenst egy körbezáró tag nélkül top-level renderelni. Most már ez lehetséges, de a szintaxias korántsem olyan tiszta és magától értetődő. Utána lehet nézni, de mi nem fogunk ezzel foglalkozni, mert nem érdemes.

Properties feltételek

  • A propok azok a komponensünk konfigurálására vannak

  • A propokat nem változtathatjuk a komponensen belül, erre a szó: immutable

  • Lehetnek stringek

    <User name="Jane" title="CEO" />

  • Más típusokhoz kapcsos zárójeleket kell használnunk

    <User name="Jane" salary={ 100000 }
    hobbies={ ["bridge", "reading", "tea"] } />

Propok használata

  • this.props.propertyName
  • Nem változtathatjuk, csak megadhatunk újakat!!

Kondicionálisak JSX-ben

A render() metódus visszatérhet:

  • egyetlen valid DOM objektumként (return <div>...</div>)
  • DOM objektumok tömbjeként (ezt még hagyjuk egyelőre!)
  • null (undefined az nem oké!)

Akármilyen logikát használhatunk a render() metódusban:

class Lottery extends React.Component {
  render() {
    if (this.props.winner)
      return <b>You win!</b>;
    else
      return <b>You lose!</b>;
  }
}

Ternary

class Lottery extends React.Component {
  render() {
    return (
      <b>You {this.props.winner ? "win" : "lose"}!</b>
    )
  }
}

Looping in JSX

Nagyon gyakran array.map(fn) fg-t használunk, hogy egy tömb elemeit kirajzoljuk:

class Messages extends React.Component {
  render() {
    const msgs = [
      {id: 1, text: "Greetings!"},
      {id: 2, text: "Goodbye!"},
    ];

    return (
      <ul>
        { msgs.map(m => <li>{m.text}</li>) }
      </ul>
    );
  }
}

React stilizálása

CSS classokat kaphatnak a JSX tagek.

Mivel a class egy lefoglalt kulcsszó a Reactban, ez className szóval helyettesztendő:

class Message extends React.Component {
  render() {
    return <div className="urgent">Emergency!</div>
  }
}

Inline is használhatunk stílusokat, ilyenkor a style egy JS objektumot vár:

class Box extends React.Component {
  render() {
    const colors = {
      color: this.props.favoriteColor,
      backgroundColor: this.props.otherColor,
    };

    return <b style={colors}>{this.props.message}</b>;
  }
}
